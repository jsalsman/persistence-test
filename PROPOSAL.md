### Proposal: The Guild.ai LLM-Powered Auto-Changelog Generator

**The Oath:**
The `repo-change-summarizer` automatic change log generator assumes the responsibility of maintaining clear, human-readable documentation of repository activity. Its primary purpose is to eliminate the manual toil of writing changelogs by automatically parsing GitHub push events and appending formatted summaries to a `GUILDAI_CHANGELOG.md` file. Crucially, it provides semantic context—AI-generated summaries explaining the *why* and *what* of actual code changes—that the native GitHub user interface completely lacks, aggregating this invaluable information into a single, easily scannable file. This agent is designed for any developer or engineering team utilizing Guild.ai who wants an autonomous, self-documenting workflow without having to configure external, heavy-duty CI/CD pipelines.

**The Reagents:**
To function properly, this agent requires a workspace configured with a GitHub Integration that possesses "Read & Write" access to Repository Contents. Internally, it relies entirely on the native `@guildai/agents-sdk`, specifically utilizing `gitHubTools` for fetching commit patches (`github_repos_get_commit`) and managing file states (`github_repos_create_or_update_file_contents`). It also requires environment access to the undocumented `task.llm.generateText` capability to dynamically synthesize summaries. It does not require persistent VM access or code-execution sandboxes, as it operates entirely via a headless GitHub Event Trigger bound to the `push` event.

**The Ritual:**
The agent executes via a highly defensive, ES5-compliant execution loop engineered to run safely within restrictive AST compiler environments. Upon invocation by a webhook, it dynamically extracts the routing coordinates (repository owner and name) directly from the payload. It iterates through the payload's commits, fetching the specific Git `.patch` data for each file. To conserve LLM token context and reduce documentation noise, it prompts the AI to summarize the full text of *new* files, but restricts the AI to summarizing only the *diff patch* for modified files. It employs explicit infinite-loop protection by silently ignoring changes made to the changelog itself, ultimately compiling the summaries and committing them back to the repository.

**The Proof:**
Success is definitively measured by observing the target repository's `GUILDAI_CHANGELOG.md` file following a developer's push event. A fully operational agent will successfully intercept the webhook, process the diffs, and seamlessly append a beautifully formatted Markdown entry containing the pusher's username, a timestamp, and exactly one concise, AI-generated sentence describing the semantic purpose of each added or modified file.
